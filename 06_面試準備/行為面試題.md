# 💬 行為面試題準備（BQ - Behavioral Questions）

> 行為面試考的是你過去的經歷和處事方式，不是技術知識

---

## 📌 為什麼需要準備 BQ？

```
外商和大公司都會問 BQ，原因：
1. 技術可以學，但性格和工作態度難改
2. 了解你如何處理困難和壓力
3. 確認你能和團隊合作
4. 預測你未來的表現

即使是韌體工程師職位，BQ 也可能佔面試 30-40%！
```

---

## 🔷 回答技巧：STAR 法則

```
S - Situation：背景情況
T - Task：你的任務/責任
A - Action：你採取的具體行動
R - Result：結果和學習

範例問題：「說一個你解決困難問題的經驗」

「在我之前的專案中，我們發現 sensor 讀取偶爾會失敗（S）。
我負責找出問題並修復（T）。
我用示波器分析 I2C 波形，發現是 clock stretching 太長，
然後修改了 driver 的 timeout 設定，並加入了 retry 機制（A）。
問題解決後，sensor 讀取的成功率從 95% 提升到 99.9%（R）。」
```

## 🔷 常見 BQ 問題和準備

### 1. 自我介紹

```
問法：
- Tell me about yourself
- 請簡單介紹一下你自己

準備要點：
- 1-2 分鐘
- 包含：背景、現職、為什麼想加入

範例：
「我是 [名字]，[學校] 資工碩士畢業。
目前在 [公司] 擔任韌體工程師，主要負責 OpenBMC 的開發，
包括 sensor 整合和系統除錯。
我對 BMC 和嵌入式系統有濃厚興趣，
想加入 [目標公司] 是因為你們在伺服器管理領域有領先地位，
而且我看到 [具體原因，如產品或技術]。」
```

### 2. 為什麼想離開現在的公司？

```
問法：
- Why are you looking for a new job?
- 為什麼想離開？

⚠️ 避免：
- 抱怨前公司或主管
- 說薪水太低（除非對方主動問）
- 負面情緒

✓ 建議：
「我在現職學到很多 OpenBMC 的基礎，
但我希望能接觸更多挑戰性的專案和更先進的技術。
[目標公司] 在 [具體領域] 的技術讓我很嚮往，
我相信能在這裡繼續成長。」
```

### 3. 你最大的成就是什麼？

```
問法：
- What's your proudest achievement?
- 你最有成就感的事是什麼？

準備：選一個展現技術能力 + 問題解決的案例

範例：
「我最有成就感的是解決了一個客戶回報的 live issue。
當時 BMC 在特定情況下會 crash，影響到生產線。
我花了兩天分析 log 和 coredump，發現是 race condition，
在兩個服務同時存取 D-Bus 物件時發生。
我設計了一個解決方案，加入適當的鎖機制，
最後成功修復並上線，客戶非常滿意。」
```

### 4. 你最大的失敗/挫折是什麼？

```
問法：
- Tell me about a failure
- 你遇過最大的挫折是什麼？

⚠️ 避免：
- 說「我沒有失敗過」
- 說影響太嚴重的錯誤
- 把責任推給別人

✓ 建議：選一個能展現你從失敗中學習的案例

範例：
「有一次我急著趕進度，code review 沒仔細看就 merge 了，
結果造成了 regression bug，影響了其他功能。
從那次之後，我學會了不管多趕都要認真 review，
也開始推動團隊增加自動化測試覆蓋率。」
```

### 5. 如何處理和同事的意見不同？

```
問法：
- Tell me about a conflict with a coworker
- 你如何處理意見分歧？

範例：
「曾經和同事對 API 設計有不同看法。
我先聽完他的理由，然後說明我的考量。
我們發現各自有不同的優先考慮：他著重擴展性，我著重效能。
最後我們找主管討論，決定採用折衷方案，
並把兩種考量都納入設計原則。
這件事讓我學到，不同觀點可以讓設計更完整。」
```

### 6. 你的優點和缺點？

```
問法：
- What are your strengths and weaknesses?
- 你的優缺點是什麼？

優點：選和工作相關的
「我的優點是 debug 能力強，喜歡追根究柢找出問題本質。
在之前的工作中，我常常被指派處理困難的 bug。」

缺點：說一個真實但可以改進的
「我有時候會太專注在細節上，花太多時間優化不重要的地方。
我正在練習先完成核心功能，再回來優化。」

⚠️ 避免說「太認真」「太努力」這種假缺點
```

### 7. 為什麼選我們公司？

```
問法：
- Why do you want to join us?
- 為什麼想加入 [公司]？

準備要點：
- 研究公司和產品
- 找到真正吸引你的點
- 連結到你的目標

範例（NVIDIA）：
「我選擇 NVIDIA 有幾個原因。
第一，NVIDIA 的 GPU 伺服器是 AI 基礎設施的核心，
能參與這樣的產品讓我很興奮。
第二，我看到 NVIDIA 在 OpenBMC 社群很活躍，
這代表團隊重視開源和技術分享，這是我認同的價值。
第三，我希望在更 scale 的環境下挑戰自己。」
```

### 8. 你對工作的期望是什麼？

```
問法：
- What are you looking for in your next role?
- 你對工作的期望是什麼？

範例：
「我希望能做有技術深度的工作，
繼續深耕 BMC 和嵌入式系統領域。
同時我也希望能有學習和成長的空間，
接觸到更先進的技術和更大規模的系統。
另外，我看重團隊的合作氛圍，希望能在開放的環境中工作。」
```

### 9. 五年後你想在哪裡？

```
問法：
- Where do you see yourself in 5 years?
- 五年後你想做什麼？

範例：
「五年後我希望成為 BMC 領域的專家，
能夠主導重要功能的設計和開發。
我也希望能 mentoring 新人，分享我的經驗。
長期來說，我對 tech lead 或架構師的角色有興趣。」

⚠️ 不用說得太具體，但要展現成長意願
```

### 10. 你有什麼問題想問我們？

```
準備 3-5 個問題：

關於工作內容：
- 這個職位日常工作大概是什麼？
- 團隊目前最大的技術挑戰是什麼？
- 這個職位成功的定義是什麼？

關於團隊：
- 團隊大概幾個人？分工如何？
- 團隊的工作流程是什麼樣的？

關於成長：
- 公司如何支持工程師的成長？
- 有沒有內部轉調的機會？

⚠️ 第一輪不要問薪水福利（除非對方主動提）
```

## 🔷 ADHD/GAD 相關

### 要不要揭露？

```
一般建議：面試時不需主動揭露

如果需要 accommodations：
- 可以在拿到 offer 後和 HR 討論
- 許多外商對 neurodiversity 友善
- NVIDIA、Google 等都有相關政策

面試時可以這樣說（如果被問到工作風格）：
「我發現使用 Pomodoro 技巧和清晰的 task list 
能幫助我保持專注和高效。」
```
