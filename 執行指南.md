# 🚀 NextJob 深度擴充執行指南

> 這份文件是給**你（人類）**看的操作手冊，說明如何正確執行深度擴充任務。

---

## 檔案結構說明

```
NextJob/                            ← 專案根目錄
├── .agent/
│   ├── AGENT.md                    # 專案規則（AI 自動讀取）
│   ├── skills/
│   │   └── deep-expansion/
│   │       └── SKILL.md            # 品質標準（AI 讀取以維持深度）
│   └── workflows/
│       └── rebuild-nextjob.md      # 流程控制（用 /rebuild-nextjob 觸發）
├── 02_C語言/
├── 03_底層驅動開發/
├── 04_OpenBMC深化/
└── 05_作業系統/
```

### 各檔案的角色

| 檔案 | 角色 | 讀取方式 |
| :--- | :--- | :--- |
| `AGENT.md` | 專案規則（語言、Commit 格式） | **自動**，AI 進入專案時載入 |
| `SKILL.md` | 品質標準（深度、格式） | **手動或自動**，任務相關時讀取 |
| `workflows/*.md` | 流程控制（順序、Commit 時機） | **手動**，用 `/workflow-name` 觸發 |

---

## 執行方式

### 方法 A：使用 Workflow（推薦，最簡單）

1. **開一個新 Conversation**
2. 確保 AI 的工作目錄是 NextJob 專案根目錄
3. 輸入：
   ```
   /rebuild-nextjob
   ```
4. AI 會自動讀取 Workflow 和 Skill，依序執行

---

### 方法 B：直接下指令

1. **開一個新 Conversation**
2. 確保 AI 的工作目錄是 NextJob 專案根目錄
3. 貼上：
   ```
   我要深度擴充這個專案中的所有學習文件。

   執行規則：
   1. 先閱讀 `.agent/skills/deep-expansion/SKILL.md` 了解深度標準
   2. 按順序擴充：02_C語言 → 03_底層驅動 → 04_OpenBMC → 05_作業系統
   3. 每完成一個「檔案」後，報告進度並等我說「繼續」
   4. 每完成一個「目錄」後，執行 Git Commit

   現在開始，從 02_C語言/ 的第一個檔案開始。
   ```

## 品質監控

### 如果 AI 開始偷懶

直接說：
```
這個章節寫得太淺了。
請重新閱讀 `.agent/skills/deep-expansion/SKILL.md`，然後重做這個章節。
```

### 品質檢查點

每個章節完成後，檢查：
- [ ] 行數是否達到 1500+ 行？
- [ ] 是否包含 Why/What/How/Deep Dive 結構？
- [ ] 是否有至少 10 題面試題？
