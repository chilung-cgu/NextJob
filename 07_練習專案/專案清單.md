# 🛠️ 練習專案清單

> 透過實作加深理解，同時可以放在履歷/GitHub 上

---

## 📌 專案建議層級

```
Level 1：入門（確認基礎）
Level 2：進階（展示能力）
Level 3：挑戰（脫穎而出）

建議順序：先完成 L1，確保基礎穩固，再挑選 L2 深入。
```

---

## 🔷 Level 1：入門專案

### 專案 1-1：C 語言面試題練習庫

```
目標：把面試常考的 C 題目都實作一遍

內容：
- 實作常見資料結構（Linked List, Stack, Queue）
- 寫 bitwise 操作函式
- 練習指標和記憶體管理

產出：
- GitHub repo 放你的練習程式碼
- 每題有註解說明

時間：1-2 週
```

### 專案 1-2：I2C Sensor 模擬器（Linux）

```
目標：了解 Linux I2C 子系統

內容：
- 在 Linux 上用 i2c-tools 讀寫
- 閱讀一個真實 sensor 的 datasheet
- 寫一個 Python/C 程式讀取 sensor 數值

工具：
- Raspberry Pi 或任何有 I2C 的 Linux 開發板
- 或使用 i2c-stub kernel module 模擬

產出：
- 程式碼 + 說明文件
- 對 I2C 協定的理解

時間：1 週
```

### 專案 1-3：Linux Kernel Module 入門

```
目標：了解 Linux driver 基礎

內容：
1. 寫一個 hello world kernel module
2. 實作一個簡單的 character device
3. 讀懂 module_init / module_exit 流程

產出：
- 可載入的 .ko 檔案
- Makefile
- README 說明

時間：1 週
```

## 🔷 Level 2：進階專案

### 專案 2-1：仿 hwmon 的 Sensor 監控程式

```
目標：理解 OpenBMC hwmon 的運作

內容：
1. 寫一個 daemon 週期讀取 /sys/class/hwmon
2. 將 sensor 數值透過 stdout 或 log 輸出
3. （進階）加入 threshold 告警機制

技術：
- C/C++ 程式設計
- Linux sysfs 操作
- 定時器或 poll 機制

產出：
- 完整的 daemon 程式
- systemd service 檔案
- 說明文件

時間：2 週
```

### 專案 2-2：D-Bus Service 實作

```
目標：理解 D-Bus IPC 機制

內容：
1. 寫一個簡單的 D-Bus service
2. 實作一個 method 和一個 property
3. 用 busctl 測試

技術：
- C++ 或 Python
- sd-bus 或 python-dbus
- D-Bus introspection XML

產出：
- 可運行的 D-Bus service
- 說明文件

時間：2 週
```

### 專案 2-3：簡易 UART Boot Console

```
目標：理解 UART 和 boot 流程

內容：
1. 用 MCU（如 Arduino/STM32）做一個 UART 介面
2. 模擬 boot 流程的訊息輸出
3. 在 PC 上用 minicom/screen 連線觀看

技術：
- UART 協定
- 嵌入式 C
- Python（host 端）

產出：
- MCU 韌體
- Host 端工具
- 說明文件

時間：1-2 週
```

## 🔷 Level 3：挑戰專案

### 專案 3-1：簡易 BMC 模擬器

```
目標：展示對 BMC 架構的理解

內容：
1. 在 Raspberry Pi 或 QEMU 上模擬簡易 BMC
2. 實作基本功能：
   - 讀取 CPU 溫度
   - 控制 LED（模擬電源狀態）
   - 簡單的 REST API
3. 文件化整個架構

技術：
- Embedded Linux
- REST API（Flask/FastAPI）
- I2C/GPIO 操作

產出：
- 完整的系統映像
- API 文件
- 架構說明

時間：4-6 週

對面試的價值：
- 展示你能從零搭建系統
- 證明你理解 BMC 架構
- 面試時有具體專案可講
```

### 專案 3-2：貢獻 OpenBMC 開源專案

```
目標：真實的開源貢獻經驗

內容：
1. 在 OpenBMC GitHub 找 good first issue
2. 修復 bug 或改進文件
3. 提交 Pull Request

好處：
- 真實的貢獻記錄
- 可以放在履歷上
- 了解實際的 code review 流程

難度：
- 需要熟悉 OpenBMC 架構
- 可能需要時間等待 review

網址：https://github.com/openbmc
```

## 📝 專案執行建議

### 版本控制

```bash
# 每個專案都用 Git
git init
git add .
git commit -m "Initial commit"

# 推到 GitHub
git remote add origin git@github.com:username/project.git
git push -u origin main
```

### README 範本

```markdown
# 專案名稱

## 簡介
這個專案是什麼，解決什麼問題

## 技術
- 使用的語言/框架
- 需要的硬體/軟體

## 如何執行
1. 步驟一
2. 步驟二

## 架構
[簡要說明架構]

## 學習心得
[你從這個專案學到什麼]
```

### 專案選擇建議

```
時間有限時：
1. 優先做 Level 1 的專案確保基礎
2. 選一個 Level 2 專案深入

有更多時間時：
1. 挑戰 Level 3 的專案
2. 嘗試 OpenBMC 貢獻
```
