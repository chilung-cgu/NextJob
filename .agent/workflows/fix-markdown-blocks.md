# Polish Markdown Workflow (Previously Fix Markdown Blocks)

此 Workflow 用於依照 `markdown-polishing` Skill 的標準，全面優化 `.md` 文件的格式。不僅修復程式碼區塊錯誤，還包含表格、列表與問答格式的標準化。

## 步驟

1.  **確認目標**：
    - 檢查使用者指定的檔案，若無指定則檢查當前開啟的檔案。

2.  **執行 Markdown 優化掃描 (參考 `markdown-polishing` Skill)**：

    - **[Table] 檢查與修復表格**：
        - 移除包圍在 Markdown 表格外的 ` ``` ` 代碼區塊符號。
        - 將純文字繪製的 ASCII 表格（使用 `+`, `-`, `|` 符號）轉換為標準 Markdown 表格語法。

    - **[List] 檢查列表階層**：
        - 掃描編號列表與項目符號。確認子項目（Sub-items）是否有足夠的縮排（建議 4 空格）。
        - 將手動編號如 `(1)`, `(2)` 在非行內文提及的情況下，轉換為標準 Markdown 有序列表 (`1.`, `2.`)。

    - **[Q&A] 優化問答格式**（若為面試或 FAQ 文件）：
        - 確保問題 (**Q**) 使用粗體。
        - 確保回答 (**A**) 位於下一行，並使用列表符號 (`*` 或 `-`) 縮排，形成清晰的視覺階層。
        - 範例：
            ```markdown
            **Q: 問題描述**
            *   **A**: 回答內容
            ```

    - **[Code] 檢查程式碼區塊**：
        - 確認所有 ` ``` ` 區塊都有標註語言（如 `bash`, `cpp`, `json`）。

3.  **執行修復**：
    - 針對上述掃描到的問題進行內容替換。
    - 保持原意不變，僅調整 Markdown 語法標記。

4.  **驗證**：
    - 確認修改後的 Markdown 能正確渲染：表格有框線、列表階層正確縮排、Q&A 重點分明。
